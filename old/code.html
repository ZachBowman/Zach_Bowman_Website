Programming / Code



Aside from a few XBOX-specific code snippets, like the file saving system, everything in Blockhead was programmed entirely on my own, from the animations to the game logic to the menu screens. This represents the most comprehensive showcase of my abilities thus far.
Particle System
One of the most useful graphical tools I built for Blockhead is a comprehensive 2D particle system which has already received some significant upgrades in my newest project, Boxland. The functions can be used to simulate almost anything, including smoke, fireworks, laser beams or explosions just by tweaking the values and supplying a few images as a base. The programmer can adjust the amount, speed, directions, spread, transparency, and fade speed of the effects. It was programmed for XNA Game Studio, but can be easily ported to any C-based platform.
The Boxland engine represents the first evolutionary step in a system I'm building from the ground up. On the surface, the game is tile-based and from a 2D overhead perspective, but the game world is actually running in 3D. All of the surfaces are kept track of in terms of brushes and textures (like an early Quake or Half-Life engine), and the characters and objects have three-dimensional locations and physics. The advantage to building the engine from scratch is that I will be able to create environments that are not only completely destructible, but with a level of detail and interactivity that we have rarely seen in games before. Subsequent projects built on this code will include 3D graphics, map editor compatibility, and support for any game style, from first-person shooter to role-playing.
Dreamscape
Closely tied to the game engine outlined above is a companion tool called Dreamscape. Dreamscape will be a map editor that will allow the designer to incorporate randomly generated elements into the game's design, creating an experience that changes each time a new game is started; the ultimate in replay value. The game-created details would be as simple as enemies randomly placed within locations designated by the designer, or as complex as complete areas of the game world generated by the game engine's ai. Eventually, my goal is to achieve a system in which large portions of the game are created by the game itself based on parameters set by the user. In essence, we would be teaching a game how to make new games.
