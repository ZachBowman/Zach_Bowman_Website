Word Wars

This was an idea for a supercharged, scrabble-like spinoff game I pitched, and then assembled a team to help develop. Working together, we used jQuery UI, a dictionary API, and data binding we wrote ourselves to make an html web page into a fully interactive board game (using no game engines or canvas tags). It supports up to four players, and features multiple board shapes, dictionary definitions, and a fully functional validation and scoring system.

HTML5 Games

Between larger projects, I'm currently developing a series of browser-based minigames in HTML5. The goal is to start small and push the limits of Javascript in terms of graphics, gameplay complexity and user interface to test how far it can take us. My first two minigames are now complete, with several more planned, including arcade, platformer, action, and puzzle games.

All my 2D browser games use a custom, responsive, canvas framework I built in-house to support sprite rendering, user input, sound, and music across all desktop and mobile browsers.
